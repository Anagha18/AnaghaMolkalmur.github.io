MarvinEye

A multimodal, flexible and robust computer vision framework for constrained environment autonomy

Concepts Used: Polynomial Regression, Computer Vision, GPU Acceleration

×

MarvinEye

Summary. MarvinEye gives the oject it is integrated with a sense of position by estimating distance to a pre-set visual aid, in a resource optimal manner. The DSMAC termina way guidance algorithm for air artillery systems has been adapted to fit this framework and goes by the name SDEC, where the distance moved is validated based on how much of the frame the object occupies- Clarity of Vision (CoV). The SDEC derives an equation for the best fit curve plotted through the CoV, using Polynomial Regression. The Sensor height and Triangle Similarity algorithms have also been used, after optimizing the same. In order to increase the throughput, the code has also been accelerated on the GPU. The MarvinEye framework can be integrated into white canes for the blind, path following bots and other constrained environment cases. It can also be integrated into Rovers to ensure smooth operations. have authored the paper and submitted it to the International Conference on Innovations in Information Technology 2020 to be held in the UAE.

SoLoMoBro

A Fault Tolerant Mobile Ad-hoc network that distributes tasks in task level parallelism.

Concepts Used: MANET, Fault Tolerance, Device Heterogentiy, Load Balancing, TCP/IP

×

SoLoMoBro


Summary.Designed and Implpemented a MANET in a plug and play fashion that distributes tasks to clients in the cluster over TCP/IP, over a star topology with 1 master node and several clients. It is android application that can be installed on mobile devices and also compatible on linux systems, it functions over the LAN. Any user can assign work through the server, the tasks are partitioned and distributed over the client cluster based on priority of the client using weighted round robin load balancing, taking the device heterogenity into consideration (utilizing client specs like number of cores and clock speed currently available). A client can join the network at any point in time, the network handles both master node and client faults and gets the network functioning again in a very short duration. I have authored the paper and submitted it to the Internation Conference on Distributed Computing and Internet Technology 2021 to be held in Orissa, India.
Language Used. C++, C, CMake, Java.

RideShare


Summary.Created a Ride Sharing Application utilizing the Amazon Web Services EC2 instance. The application was hosted on Amazon EC2 instance and an SQLite Database was used to store data. The application provided services to create a user, ride, add cab sharers to the ride, cancel a ride, delete a user. MPCA

A traffic density based traffic controller and pedestrian safety model

Concepts Used: Arduino Uno, Embedded C, Obstruction Sensors, LDR

×

MPCA


Summary. Designed and Implemented a model using Arduino in Embedded C. The model made was taking into account the amout of traffic congestion in cities as well as stray animals that may unexpectedly corss roads. The model is a 4-way traffic junction that has 4 signals placed individually on all sides. 2 infrared sensors were placed at a measured distance on either side of the road for each of the 4 roads. LED lights were set up to imitate traffic lights. The input for these LEDs were from the infrared sensors, from its assigned road. If the traffic density on a particular road (measured by infrared) was high ( a lot of cars one behind the other), the light would switch to green and remain green till the traffic reduced. If all have equal density, the LEDs emulate a timed Traffic Signal in a pre-determined order. The Ultrasonic Sensors are places at the Zebra Crossing and they can distinguish between a car and a person. This is done by using 2 ultrasonic sensors, and also taking the current traffic light status into consideration. If the Light is green and there is no vehicle in the vicinity, the ultrasnic does not trigger the sound alert system. The Sound alert system is triggered if a pedestrian is crossing the road when a car is speeding at a certain distance from the pedestrian. The sound system acts as a warning to the driver, alerting him/her of the pedestrian. The Sound system is also not triggered when the light is red and the cars have stopped s=to let the pedestrian pass. The LDRs were used in the project to automatically turn the street light LEDs on at night fall.
Technology used. Arduino, Embedded C, Obstacle Sensors, Sound Actuators, LDRs

Libment

A multi-pupose and multi-view Library Management System

Concepts Used: SQL, MySQL, PostgreSQL, JavaScript, PHP, HTML, CSS

×

Libment


Summary. Designed the System starting from the ER Diagram, then coming up with the relational Tables. It was later converted into MySQL for ease of hosting onto a website. The Library Management System consisted of 3 views- the manager view, the Librarian View, the Member View. All 3 can view the list of book available across all branches and search through the same. The members can additionally reserve books. The librarian can issue the books and return them for the members through this online portal. The Librarian can also add members and remove members based on the member's duration and renewal of membership. The manager can add and remove librarians in any branch. All these were stored in PostreSQL and MySQL and hosted with a user friendly front-end using HTML, CSS, JavaScript integrated design. PHP served as the backend. The user can execute simple and complex queries, using joins, on the database through the website, such as searching for a book given a book name or Author or even a few letters that come within the book name or Author Name.
Technology used. MySQL, PostgreSQL, HTML, JavaScript, PHP, CSS, Apache Server.

Toy Compiler

A mini compiler for Java in C

Concepts Used: AST, CO, ICG, ARM, Yacc, Lex, Python

×

Toy Compiler


Summary. Created a mini compiler for Java written in yacc and Lex, to initially generate the Abstract Syntax Tree from a Java class including constructs like multiple nested for loops. Then the AST was used as an input for the Intermediate Code Generation in the form of 3 address codes. This was later taken as input for code optimization. The code optimization removed dead code, x and y. The code was then pre-processed before converting the optimised code to ARM Assembly. The pre-processing was automating the accumulation of commands under their respective branches. From here, the Assembly code generated was ARM, this was again dont through python and all arithmetic, logical and relational operations conditions were taken care of as well. This whole mini compiler execution was automated through bash scripts.
Technology used. Yacc, Lex, Shell Scripting, Python, Java.
